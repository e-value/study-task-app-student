# Lesson4: Serviceは本当に便利か？〜振り返りと違和感を言語化する〜

## 🎯 この Lesson の目的

この Lesson4 では **新しい実装は行いません。**

これまでの Lesson を振り返りながら、以下を整理します：

1. **Lesson2 と Lesson3 の比較** - Service に切り分けたことで何が変わったか
2. **それぞれの特徴の言語化** - どんな違いがあるか、何を感じたか
3. **自分なりの気づきの発見** - 良い点、気になる点を見つける
4. **次のステップへの準備** - さらに改善できる点はないか考える

---

## 📍 Lesson4 の位置づけ

これまでの流れを振り返ってみましょう。

```
Lesson1：まずは動かす（エンドポイント設計・Controller作成）
Lesson2：Controller に全部書いてみる
Lesson3：Service / FormRequest に切り分ける
Lesson4：Lesson2 と Lesson3 を比較する ← 今ここ
```

Lesson3 までで「Controller はスリムになった」と感じた人も多いはずです。

では、Lesson2 と比較して **何が変わったのか？** を整理してみましょう。

---

## 👀 まず最初にやること（重要）

### ① ブランチを見比べる

以下のブランチを **コードを書かずに眺めてください**。

-   `lesson2` または `lesson2_自分の名字`
-   `lesson3` または `lesson3_自分の名字`

📌 ポイント

**コードを正確に理解する必要はありません。**

次の点だけを意識してください：

-   Controller には何が残っているか
-   Service にはどんな処理が集まっているか
-   Service のメソッド名・private メソッド名は何を表しているか
-   どのファイルが読みやすいと感じるか

**重要：**

> 「どちらが良い・悪い」を決めつけるのではなく、
> **「何が変わったか」「それぞれの特徴は何か」** を観察してください。

---

## 💡 このレッスンで体験すること

Lesson4 は **実装ではなく振り返り** が中心です。

以下の観点で、Lesson2 と Lesson3 を比較してください：

### 比較の観点

1. **コードの見通し**

    - どちらが読みやすいか？
    - どちらが変更しやすそうか？

2. **責務の明確さ**

    - Controller の役割は明確になったか？
    - Service の役割は明確か？

3. **再利用性**

    - 処理を別の場所で使い回せそうか？
    - テストは書きやすそうか？

4. **その他、気づいたこと**
    - どんな感想を持ったか？
    - どちらの書き方が好みか？（正解はありません）

### ① それぞれの特徴を書き出す

以下の観点で **メモを取ってください**（箇条書きでOK）：

**Lesson3（Service に切り分ける）について：**
-   どんな特徴があるか
-   読みやすい点・読みにくい点
-   変更しやすそうか・しにくそうか

### ② 気づいたことを書き出す

以下の観点で **メモを取ってください**（箇条書きでOK）：

-   Lesson2 から Lesson3 で何が変わったか
-   どちらが好みか（理由も）
-   実際に開発するならどちらを選ぶか
-   さらに改善できそうな点はあるか

---

### ③ 自分なりの「疑問」を作る

正解を出す必要はありません。

例：

-   どちらの書き方が良いのか？
-   状況によって使い分けるべきか？
-   Service に切り分けたけど、次はどうすべき？
-   他にも改善の方法はあるか？

**「わからない」状態をそのまま言葉にすることが重要です。**

### それぞれのアプローチの特徴を理解する

**Lesson2（Controller に全部書く）：**

-   すべてのコードが1ヶ所にある
-   処理の流れが追いやすい
-   （あなたが感じた特徴を追加してください）

**Lesson3（Service に切り分ける）：**

-   Controller がスリム
-   責務が分離される
-   （あなたが感じた特徴を追加してください）

## 🎤 対面ディスカッションに向けて

Lesson4 の内容は、**このあと対面でディスカッションします。**

ディスカッションでは、

-   それぞれのアプローチをどう感じたか
-   他のメンバーはどう感じたか
-   どんな改善の方法が考えられるか
-   実際のプロジェクトならどう選ぶか

をテーマに話し合います。

**準備しておくこと：**

-   自分が感じた「Lesson2 の特徴」のメモ
-   自分が感じた「Lesson3 の特徴」のメモ
-   「これはどうすべき？」という疑問のリスト
